## METHOD TO THE MADNESS
* Plan the problems using the neetcode roadmap
* HAVE THE TIMER READY
* Set the timer to 10 minutes
* 10 minute window consists of:
    * UNDERSTAND THE PROBLEM BY:
        * SCAN THE PROBLEM
        * looking at keywords to the problem
        * check the examples below
        * check the constraints
    * PLAN THE APPROACH in paragraph form
    * Determine Time and Space Complexity 
    * Solve the problem
* After the 10 minute window:
    * Did I solve the problem?
    * Can we compare the current problem with other previous problems?
    * Do we have a base problem?
    * What is the difference between problems?
    * if not,
        * check test cases in remarkable
        * Practice print statements to see how can I effectively debug the problem with the help of Gemini
        * ask questions based on gaps
        * identity gaps
    * if I did solve the problem:
        * Optimize
        * Refactor
    * Reflect on what I learned and identify the gaps wihtin my code
    * write implementation in paragraph form
    * ask questions
* Create a running list of python functions, tips and tricks so we can just refer to one document 
* REDO ON THE SAME NOTEBOOK
* For intuition, start out with smaller test cases
